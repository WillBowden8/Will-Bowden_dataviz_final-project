In this project I explore two datasets: Tampa International Airport (TPA) weather data from 2022 for distribution visualization exercises, and a concrete compressive strength dataset for a multi-part exploratory analysis. ### (e) Precipitation Distribution — Custom Visualization

Values of −99.99 represent missing precipitation data and are excluded.


``` r
precip_clean <- weather_tpa |>
  filter(precipitation > -99) |>
  # treat trace precipitation (0) separately by adding a small offset for log scale
  mutate(precip_plot = if_else(precipitation == 0, 0.001, precipitation))

# Proportion of dry days
dry_pct <- round(mean(weather_tpa$precipitation[weather_tpa$precipitation > -99] == 0) * 100)

ggplot(precip_clean, aes(x = precip_plot, fill = after_stat(x))) +
  geom_histogram(bins = 35, color = "white", linewidth = 0.2) +
  scale_x_log10(
    breaks = c(0.001, 0.01, 0.1, 0.5, 1, 2, 4),
    labels = c("0", "0.01", "0.1", "0.5", "1", "2", "4")
  ) +
  scale_fill_viridis_c(option = "viridis", guide = "none") +
  annotate(
    "text", x = 0.001, y = Inf,
    label = paste0(dry_pct, "% of days\nhad no rain"),
    hjust = 0, vjust = 1.3, size = 3.8, color = "gray30"
  ) +
  labs(
    title    = "Daily Precipitation at Tampa International Airport, 2022",
    subtitle = "Log-scaled x-axis; leftmost bar represents zero-precipitation days",
    x        = "Precipitation (inches, log scale)",
    y        = "Number of Days",
    caption  = "Source: FSU Florida Climate Center · Missing values (−99.99) excluded"
  ) +
  theme_minimal(base_size = 13) +
```

## PART 2: Data on Concrete Strength

Concrete is the most important material in **civil engineering**. Concrete compressive strength is a highly nonlinear function of age and ingredients. The dataset is from the [UCI Machine Learning Repository](https://archive.ics.uci.edu/ml/index.php) and contains 1,030 observations with 8 input variables and 1 output variable.

| Variable | Notes |
|---|---|
| Cement | kg in a m³ mixture |
| Blast Furnace Slag | kg in a m³ mixture |
| Fly Ash | kg in a m³ mixture |
| Water | kg in a m³ mixture |
| Superplasticizer | kg in a m³ mixture |
| Coarse Aggregate | kg in a m³ mixture |
| Fine Aggregate | kg in a m³ mixture |
| Age | in days |
| Concrete compressive strength | MPa (megapascals) |

### Task 3 — Interactive Scatterplot: Cement vs. Compressive Strength

> **Interactivity:** This plotly chart lets readers hover over individual data points to see exact ingredient values (cement, water, age), zoom into dense regions, and filter by strength quartile via the legend. A static scatterplot of 1,030 overlapping points would obscure the structure that becomes visible when exploring with hover and zoom.


``` r
# Build strength quantile groups for color
concrete_plot <- concrete |>
  mutate(
    strength_q = cut(
      Concrete_compressive_strength,
      breaks = quantile(Concrete_compressive_strength, probs = seq(0, 1, 0.25)),
      labels = c("Q1 (lowest)", "Q2", "Q3", "Q4 (highest)"),
      include.lowest = TRUE
    )
  )

p_conc <- ggplot(
  concrete_plot,
  aes(
    x     = Cement,
    y     = Concrete_compressive_strength,
    color = strength_q,
    size  = Age,
    text  = paste0(
      "Strength: ", round(Concrete_compressive_strength, 1), " MPa<br>",
      "Cement: ",   round(Cement, 1), " kg/m³<br>",
      "Water: ",    round(Water, 1), " kg/m³<br>",
      "Age: ",      Age, " days<br>",
      "Quartile: ", strength_q
    )
  )
) +
  geom_point(alpha = 0.65) +
  scale_color_viridis_d(option = "plasma", name = "Strength Quartile") +
  scale_size_continuous(range = c(1, 6), name = "Age (days)") +
  labs(
    title = "Cement Content vs. Compressive Strength",
    x     = "Cement (kg / m³)",
    y     = "Compressive Strength (MPa)",
    caption = "Source: UCI ML Repository · concrete.csv"
  ) +
  theme_minimal(base_size = 13) +
  theme(panel.grid.minor = element_blank())

ggplotly(p_conc, tooltip = "text") |>
  layout(
    title = list(
      text = paste0(
        "Cement Content vs. Compressive Strength",
        "<br><sup>Hover for values · point size = age in days · color = strength quartile · click legend to filter</sup>"
      )
    ),
    legend = list(orientation = "h", y = -0.2)
  )
```

## Redesign: Before & After

### The "Bad" Chart — Pie Chart of NBA Championship Shares

A pie chart is a classic example of a chart that makes comparison unnecessarily difficult. Below is a reproduction of how NBA championship shares (1980–2018) might be displayed as a pie chart.


``` r
nba_raw <- read_csv("../data/NBAchampionsdata.csv", col_types = cols())

nba_raw <- nba_raw |>
  mutate(Team = str_trim(str_remove_all(Team, "'")),
         Team = if_else(Team == "Warriorrs", "Warriors", Team))

champs_pie <- nba_raw |>
  distinct(Year, Team) |>
  count(Team, name = "championships", sort = TRUE)

# Bad chart: pie with rainbow palette
pie(
  champs_pie$championships,
  labels  = paste0(champs_pie$Team, " (", champs_pie$championships, ")"),
  col     = rainbow(nrow(champs_pie)),
  main    = "NBA Championship Shares (1980–2018)"
)
```

**What is wrong with this chart?**

1. **Pie charts are hard to compare** — Human perception is poor at judging arc length and angle. It is nearly impossible to tell whether the Spurs (5) or Pistons (3) piece is larger without looking at the labels.
2. **Rainbow palette is not colorblind-safe** — The rainbow scale cycles through colors that are indistinguishable under deuteranopia or protanopia.
3. **No natural ordering** — The slices have no inherent sequence, so the viewer must search around the pie to find a team.
4. **Overplotting of labels** — Some team labels overlap, especially for smaller slices.

**How the redesign fixes the problems:**

1. **Bar length is easy to compare** — Bar charts use a common baseline and linear length, which humans perceive accurately. The Lakers' dominance (10 titles) and the gap between the Bulls (6) and everyone else is immediately visible.
2. **Colorblind-safe palette** — The viridis plasma gradient is perceptually uniform and distinguishable under common forms of colorblindness. Color is redundant here (position already encodes magnitude), so it provides visual polish without introducing ambiguity.
3. **Ordered by value** — `fct_reorder()` ranks teams from most to fewest championships, so the chart reads as a ranking.
4. **Direct labels** — Championship counts are placed at the end of each bar, eliminating any need to look up a legend or estimate from an axis.

## Summary

This project explored two datasets using a range of visualization techniques:

- **TPA weather data (2022):** Reproduced five distribution charts — faceted histograms, kernel density plots, and a ridge plot with quantile lines — demonstrating how the choice of geometry and bandwidth reveals different aspects of a temperature distribution. The precipitation visualization shows that log scaling is essential when data span several orders of magnitude.

- **Concrete strength data:** Explored two continuous variables to confirm realistic data ranges, used violin + box plot combinations to show how compressive strength accumulates with curing age, and created an interactive scatter plot linking cement content to strength (colored and sized by quartile and age).

- **Accessibility:** All figures use colorblind-safe palettes (Okabe-Ito, viridis/plasma) and include `fig.alt` alt text. Shape and position encode the primary comparisons, with color used for reinforcement rather than as the sole encoding channel.

- **Redesign:** The pie chart → bar chart redesign illustrates how a bad chart can mislead through perception limits, unsafe palettes, and lack of ordering — and how a simple horizontal bar chart, properly ordered and labeled, corrects all of those problems simultaneously.
